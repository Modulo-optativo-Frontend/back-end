# ADR-005: Sistema de Estilos con Tailwind CSS

## Contexto

El proyecto frontend necesita un sistema de estilos eficiente. Se evaluaron opciones:

- **CSS Modules** - Estilos encapsulados por componente
- **Styled Components** - CSS-in-JS
- **Tailwind CSS** - Utility-first CSS
- **Sass/SCSS** - Preprocesador CSS tradicional

## Decisión

Usar **Tailwind CSS v4** como framework de estilos principal.

### Justificación

1. **Desarrollo rápido**: Clases utilitarias evitan cambio de contexto CSS ↔ JS
2. **Sin nombres invented**: No hay que pensar en nombres de clases
3. **Purga automática**: Solo se incluye CSS usado en producción
4. **Consistencia**: Design tokens integrados
5. **Popularidad**: Gran ecosistema y documentación

### Configuración

```javascript
// vite.config.js
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```

```css
/* index.css */
@tailwind base;
@tailwind components;
@tailwind utilities;
```

### Enfoque de Uso

- **Clases utilitarias** directamente en JSX
- **Componentes UI reutilizables** para patrones frecuentes
- **Estilos globales** en index.css para resets y temas

## Consecuencias

### Positivas

+ **Velocidad de desarrollo**: Estilos sin salir del JSX
+ **Bundle pequeño**: Tree-shaking elimina CSS no usado
+ **Responsive fácil**: Prefijos incorporados (md:, lg:)
+ **Personalizable**: Configuración centralizada en theme
+ **Dark mode**: Soporte integrado

### Negativas

- **HTML verboso**: Clases largas en JSX
- **Curva inicial**: Aprender todas las clases
- **Diseño puede ser inconsistente** sin disciplina
- **No es CSS tradicional**: Requiere cambio de mentalidad
