# ADR-006: Gestión del Carrito de Compras

## Contexto

El e-commerce necesita gestionar el carrito de compras del usuario. Se evaluaron diferentes enfoques:

- **Carrito solo en frontend**: Estado local/Context
- **Carrito solo en backend**: Persistido en BD
- **Híbrido**: Estado local + sincronización con backend

## Decisión

Adoptar enfoque **backend-driven** con sincronización:

### Arquitectura

```
┌─────────────────┐      ┌─────────────────┐      ┌─────────────────┐
│   Frontend      │────▶│     API         │────▶│    MongoDB      │
│  (React)       │◀────│  (Express)      │◀────│   (Carrito)     │
└─────────────────┘      └─────────────────┘      └─────────────────┘
```

### Modelo de Datos

```javascript
// Carrito en MongoDB
{
  usuario: ObjectId,      // Referencia al usuario
  items: [{
    producto: ObjectId,  // Referencia al producto
    cantidad: Number     // Cantidad (mín: 1)
  }],
  createdAt: Date,
  updatedAt: Date
}
```

### Flujo de Uso

1. **Añadir al carrito**: POST /api/carrito/items
   - Requiere autenticación
   - Extrae usuario del token JWT
   - Crea carrito si no existe
   - Añade/actualiza item

2. **Ver carrito**: GET /api/carrito
   - Retorna items con datos del producto

3. **Actualizar cantidad**: PUT /api/carrito/items/:productoId

4. **Quitar item**: DELETE /api/carrito/items/:productoId

5. **Vaciar**: DELETE /api/carrito

6. **Checkout**: POST /api/pedidos/checkout
   - Convierte items del carrito en pedido
   - Congela precios unitarios
   - Vacía el carrito

## Consecuencias

### Positivas

+ **Persistencia**: Carrito survive cierre del navegador
+ **Multi-dispositivo**: Usuario puede continuar en otro dispositivo
+ **Admin visibility**: Possibility de ver carritos abandonados
+ **Checkout fiable**: Datos siempre vienen del servidor

### Negativas

- **Latencia**: Cada operación requiere request al servidor
- **Complejidad**: Más endpoints y lógica en backend
- **Desconexión**: Sin internet = sin acceso al carrito
